## 기본 형식

### 1. 파일 헤더

```markdown
# X-Y: [에피소드 제목]

**분량:** 약 4,000자  
**챕터:** X - [챕터 제목]  
**POV:** A (1인칭 현재형)

---
```

### 2. 대사/독백 형식

모든 대사와 독백은 `·` 기호로 시작합니다.

#### 독백 (A의 1인칭 시점)
```
· (독백) 눈을 뜬다. 천장이 보인다. 낡은 나무 서까래, 먼지 쌓인 대들보.
```

#### 캐릭터 대사
```
· A: "엘프 놈들... 결국 배신인가."
· B: "약속한 정령은 오지 않았습니다. 하필 최종병기 완성일에 마왕의 습격이라니, 우연일 리 없죠."
```

#### 게임플레이 요소
```
[ 전투 (리듬 게임) 진입 ]
[ 선택지 ]
```

### 3. 대사 길이

- **1줄**: 짧고 강렬한 순간
  ```
  · A: "젠장!"
  ```

- **2-3줄**: 일반적인 대사 (권장)
  ```
  · 마왕: "인간 최고의 마법사라 들었습니다만... 아직은 제 기대에 한참 못 미치는군요."
  ```

- **3줄 이상**: 감정적 정점이나 중요한 설명
  ```
  · (독백) 번지르르하게 빼입은 놈이 기분 나쁘게 사람 좋은 미소를 지으며 걸어온다. 한 걸음씩 다가올 때마다 숨이 턱턱 막힌다. 토가 쏟아질 것 같은 중압감이다.
  ```

## 분량 규칙

### 목표 분량
- **1편당 약 4,000자** (공백 포함)
- 최소 3,500자 ~ 최대 4,500자

### 분량 확인 방법

#### VS Code
1. 파일 열기
2. 하단 상태바에서 "문자 수" 확인
3. 또는 `Ctrl+Shift+P` → "Word Count" 검색

#### 온라인 도구
- https://www.charactercountonline.com/
- 공백 포함 옵션 선택

### 분량 조절 팁

**너무 짧을 때 (3,500자 미만)**:
- 독백에 감각적 묘사 추가
- 캐릭터 내면 심리 확장
- 환경/배경 묘사 추가

**너무 길 때 (4,500자 초과)**:
- 다음 에피소드로 분할 고려
- 중복 표현 제거
- 핵심만 남기고 압축

## 문체 가이드

### A의 목소리 (1인칭 독백)

#### 특징
- 거칠고 직설적
- 냉소적이면서도 예리한 관찰
- 자조적 유머
- 비속어 사용 가능

#### 좋은 예시
```
· (독백) 귀청이 떨어져 나갈 것 같은 굉음. 빌어먹을 비명소리.
· (독백) 평화롭다. 지겹도록. 아무 일도 일어나지 않는 평화.
· (독백) 이게 내 인생이다. 마왕과 맞서 싸우던 검사의 인생. 웃기지도 않는다.
```

#### 나쁜 예시 (피해야 할 것)
```
❌ · (독백) 나는 천장을 바라보았다. 낡은 서까래가 보였다.
   → 너무 평범하고 문학적. A의 목소리가 아님

❌ · (독백) 아름다운 아침이다. 새들이 지저귀고 햇살이 따스하다.
   → A는 이렇게 낭만적으로 말하지 않음

❌ · (독백) 나는 깊은 슬픔에 잠겼다.
   → 감정을 직접 설명하지 말고 보여줄 것
```

### 대사 작성 원칙

#### 1. 캐릭터별 말투 유지

**A (주인공)**
- 간결하고 직설적
- 냉소적
```
· A: "수없이 전장을 굴렀지만, '죽겠다'고 확신한 건 오늘이 처음이군."
```

**B (부하)**
- 충직하고 격식 있음
- "장군", "~습니다" 사용
```
· B: "약한 소리 마십시오, 장군. 여기서 쓰러지시면 조선은 끝입니다."
```

**다솜 (기계 정령)**
- 무뚝뚝하고 기계적
- 정확한 수치 사용
```
· 다솜: "성공 확률 23.7%. 권장하지 않음."
```

**마왕**
- 번지르르하고 예의 바름
- 기분 나쁠 정도로 공손
```
· 마왕: "반갑습니다. A. 인간 최고의 마법사라 들었습니다만..."
```

#### 2. 자연스러운 대화

**좋은 예시**
```
· 하인: "나리, 아침 식사 준비되었습니다. 오늘은 민원인 두 분이 기다리고 계십니다."
· A: "알았다. 곧 나간다. 민원이 뭔지 미리 알려줘."
· 하인: "한 분은 세금 납부 유예 요청이고, 다른 분은 이웃과의 땅 경계 분쟁입니다."
```

**나쁜 예시**
```
❌ · 하인: "나리, 식사가 준비되었고 민원인이 두 명 있습니다."
   · A: "알겠다."
   → 너무 건조하고 정보 전달만 함
```

### 리듬과 페이싱

#### 긴박한 순간 (짧고 빠르게)
```
· (독백) 젠장, 늦었다.
· (독백) 불길이 눈앞을 덮친다.
· (독백) 온몸이 타들어 간다.
```

#### 감정적 정점 (길고 시적으로)
```
· (독백) 가끔 꿈을 꾼다. 독수리 요새의 꿈. 마왕의 거대한 불덩이가 날아오고, B가 나를 업고 뛰고, 비명소리가 울려 퍼지는 꿈. 깨어나면 식은땀이 흥건하다.
```

#### 일상 장면 (중간 템포)
```
· (독백) 창밖을 본다. 작은 마을이 보인다. 초가집 몇 채, 논밭, 그리고 멀리 산. 평화롭다. 지겹도록.
```

## 리소스 정리

**모든 에피소드 끝에 반드시 포함해야 합니다!**

### 템플릿

```markdown
---

## 리소스 정리

### 배경 이미지
- **BG_이름** (설명) - 신규/재사용

### 캐릭터 스프라이트
- **캐릭터명** (표정: 목록) - 신규/재사용

### CG 일러스트
- **CG_이름** (설명) - 신규
- 없음 (CG가 없는 경우)

### 음악
- **MUSIC_이름** (설명) - 신규/재사용

### 효과음
- **SFX_이름** (설명) - 신규/재사용

### 재사용 가능한 리소스 (이전 에피소드에서)
- 목록

---

**END OF EPISODE X-Y**

**다음 에피소드 예고:**  
X-Y+1: [제목] - [간단한 설명]
```

### 예시 (episode-2-1.md 참고)

```markdown
## 리소스 정리

### 배경 이미지
- **BG_hangjik_bedroom** (한직 관아 침실) - 신규
- **BG_hangjik_corridor** (한직 관아 복도) - 신규
- **BG_hangjik_office** (한직 관아 민원실) - 신규

### 캐릭터 스프라이트
- **A** (표정: 무기력, 냉소, 공허) - 1-1에서 재사용 가능
- **하인** (표정: 공손) - 신규
- **농부** (표정: 절박) - 신규
- **상인** (표정: 불만) - 신규

### CG 일러스트
- 없음

### 음악
- **MUSIC_melancholy_daily** (우울한 일상 음악) - 신규

### 효과음
- **SFX_rooster_crow** (닭 우는 소리) - 신규
- **SFX_door_knock** (문 두드리는 소리) - 신규
- **SFX_door_open** (문 여는 소리) - 신규
- **SFX_floor_creak** (바닥 삐걱거리는 소리) - 신규
- **SFX_paper_rustle** (종이 넘기는 소리) - 신규

### 재사용 가능한 리소스 (1-1에서)
- A 캐릭터 스프라이트 (표정만 추가)
```

### 리소스 명명 규칙

#### 배경 이미지
- `BG_[장소]_[상세]`
- 예: `BG_eagle_fortress_battle`, `BG_hangjik_bedroom`

#### 캐릭터 스프라이트
- 캐릭터 이름만
- 표정은 괄호 안에 나열

#### CG 일러스트
- `CG_[장면설명]`
- 예: `CG_dasom_completion`, `CG_b_sacrifice`

#### 음악
- `MUSIC_[분위기]_[상세]`
- 예: `MUSIC_battle_urgent`, `MUSIC_melancholy_daily`

#### 효과음
- `SFX_[소리설명]`
- 예: `SFX_thunder_strike`, `SFX_door_knock`

## 체크리스트

에피소드 작성 완료 후 제출 전에 확인하세요:

### 형식
- [ ] 파일명: `episode-X-Y.md` 형식
- [ ] 헤더: 제목, 분량, 챕터, POV 포함
- [ ] 모든 대사/독백이 `·`로 시작
- [ ] 독백은 `· (독백)` 형식
- [ ] 캐릭터 대사는 `· 캐릭터명: "대사"` 형식

### 분량
- [ ] 총 3,500~4,500자 (공백 포함)
- [ ] 대사 길이 적절 (2-3줄 권장)

### 내용
- [ ] A의 1인칭 시점 유지
- [ ] 캐릭터별 말투 일관성
- [ ] 감각적이고 구체적인 묘사
- [ ] 냉소적이면서도 예리한 A의 목소리
- [ ] 전체 스토리 흐름과 일치

### 리소스
- [ ] 리소스 정리 섹션 포함
- [ ] 배경 이미지 목록
- [ ] 캐릭터 스프라이트 목록
- [ ] CG 일러스트 목록 (없으면 "없음")
- [ ] 음악 목록
- [ ] 효과음 목록
- [ ] 재사용 가능한 리소스 표시
- [ ] 신규/재사용 여부 명시

### 마무리
- [ ] "END OF EPISODE X-Y" 표시
- [ ] 다음 에피소드 예고 포함
- [ ] 맞춤법 검사
- [ ] 전체 읽어보기

## 자주 묻는 질문

**Q: 분량이 4,000자를 넘어가면 어떻게 하나요?**  
A: 4,500자까지는 허용됩니다. 그 이상이면 다음 에피소드로 분할하거나 압축하세요.

**Q: 비속어를 사용해도 되나요?**  
A: 네, A의 캐릭터에 맞다면 "빌어먹을", "젠장" 등 사용 가능합니다.

**Q: 리소스 정리에서 "재사용"은 어떻게 판단하나요?**  
A: 이전 에피소드에서 이미 사용된 리소스면 "재사용"입니다. 예: A 캐릭터 스프라이트는 1-1에서 이미 있으므로 재사용.

**Q: CG가 필요 없는 에피소드도 있나요?**  
A: 네, 일상 장면 등은 CG 없이 진행 가능합니다. "없음"으로 표시하세요.

**Q: 게임플레이 요소는 어떻게 표시하나요?**  
A: `[ 전투 (리듬 게임) 진입 ]`, `[ 선택지 ]` 같은 형식으로 표시합니다.
